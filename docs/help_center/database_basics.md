# Database Basics

## What is a Database?

A database is a collection of pages with structured properties:
- Like a spreadsheet, but more flexible
- Each row is a page
- Columns are properties
- Multiple views of same data

## Creating a Database

### Inline Database
1. Type /database
2. Select "Database - Inline"
3. Give it a title
4. Add properties

### Full-Page Database
1. Create new page
2. Select "Database" template
3. Configure properties
4. Add views

## Property Types

### Basic
- Text
- Number
- Select (single choice)
- Multi-select
- Date
- Checkbox

### Advanced
- Person (assign users)
- Files & media
- URL
- Email
- Phone
- Formula
- Relation (link databases)
- Rollup (aggregate relations)

## Views

### Table
- Spreadsheet-like
- Best for data entry
- Sort and filter columns

### Board
- Kanban-style
- Drag cards between columns
- Great for workflows

### Calendar
- Date-based view
- Shows items on calendar
- Drag to reschedule

### List
- Compact page list
- Minimal properties shown
- Fast scanning

### Gallery
- Visual cards
- Shows cover images
- Good for portfolios

## Tips

### Filtering
- Click "Filter" button
- Add conditions
- Combine with AND/OR

### Sorting
- Click column header
- Or use Sort menu
- Multiple sort levels

### Grouping
- Group by any select property
- Collapse/expand groups
- Count per group

## Common Issues

### Properties not showing
- Check "Properties" in view settings
- Ensure property has data
- Try different view type

### Slow performance
- Limit visible properties
- Use filters to reduce rows
- Archive old entries
