# Coding Task Evaluation Guide

## What to Look For

- Correct programming language as specified
- Sound logic and algorithm correctness
- Edge case handling
- Code that would actually run (syntax correctness)

## Common Failure Patterns

- **Logic Error**: Algorithm produces wrong output for valid inputs
- **Incomplete Answer**: Missing function, no error handling, partial implementation
- **Format Violation**: Wrong language, no code block formatting
- **Hallucination**: Using non-existent APIs, libraries, or language features

## Scoring Guidelines

### Instruction Following
- 5: Correct language, solves the exact problem, meets all constraints
- 3: Correct approach but misses some requirements (e.g. time complexity)
- 1: Wrong language or solves a different problem

### Reasoning Quality
- 5: Optimal or near-optimal algorithm, handles edge cases, well-reasoned
- 3: Working solution but suboptimal or misses some edge cases
- 1: Fundamentally broken logic or algorithm

### Completeness
- 5: Full solution with edge cases, error handling, and explanation
- 3: Core solution works but missing edge cases or explanation
- 1: Only skeleton code or pseudocode

### Clarity
- 5: Clean code, good naming, appropriate comments, well-structured
- 3: Working but could be better organized or documented
- 1: Unreadable or poorly structured code
