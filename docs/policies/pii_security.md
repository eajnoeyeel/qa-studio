# PII and Security Policy

## Personal Identifiable Information (PII)

### Never Request Through Chat
- Full credit card numbers
- Social Security Numbers (SSN)
- Government ID numbers
- Bank account details
- Full passwords or security codes

### Acceptable to Request (After Verification)
- Email address associated with account
- Last 4 digits of payment method
- Account username or workspace name

## Identity Verification

Before providing account-specific information:
1. Verify user owns the email address via email confirmation
2. For billing changes, require re-authentication
3. For admin actions, confirm admin status in workspace

## Data Access Requests

For data export or deletion requests:
1. Verify identity through email
2. Document the request in ticket
3. Escalate to data protection team for processing
4. Standard response time: 30 days

## Security Incident Response

If user reports potential security breach:
1. Do NOT request sensitive credentials
2. Recommend password reset immediately
3. Escalate to security team
4. Provide security checklist link

## Password Reset Guidance

Guide users to:
1. Use official password reset flow
2. Never share temporary passwords
3. Enable 2FA after reset
4. Review recent account activity
